# Components — Conventions

## Rendering mode

- **Server Component** (default, no directive): display-only, receives data as props.
- **Client Component** (`'use client'`): has state, event handlers, hooks, or browser APIs.

## File naming

- `PascalCase.tsx` — e.g. `WorkExperienceCard.tsx`, `TrackerTable.tsx`.
- Group by domain in subdirectories: `master-resume/`, `resume-editor/`, `cover-letter-editor/`, `tracker/`, `ai/`, `shared/`.

## Styling

- Use `cn()` from `@/lib/utils` to merge Tailwind classes.
- Use shadcn/ui primitives from `@/components/ui/` before writing custom markup.
- Tailwind class order: layout → spacing → sizing → typography → color → interactive.

## Data access

- Server Components may import query helpers from `src/server/queries/`.
- Client Components **must not** import Prisma or server-only modules.
- Client Components receive data as props (from a Server Component parent) or fetch via TanStack Query hooks.

## AI content color coding

- **Blue** (`text-blue-600`) = original master data
- **Orange** (`text-orange-600`) = AI-suggested content
- **Green** (`text-green-600`) = manually edited content

Apply these consistently across all editor components (resume, cover letter).

## shadcn/ui

- Components in `ui/` are auto-generated by `npx shadcn@latest add` — **do not edit manually**.
- To add: `npx shadcn@latest add [component-name]`.
- To customize, wrap the primitive in a domain-specific component.

## Component structure

- Named exports only (no default exports).
- Accept `className?: string` prop on wrapper elements.
- Complex props: define an interface above the component.
- After creating, add to the folder's `index.ts` barrel export if one exists.
